# 特定技能ドライバー 進捗管理アプリ / SSW Driver Progress Tracker

インドネシア発・特定技能ドライバーの **SSW（特定技能試験）／外国免許切替（外免）／日本語学習** の進捗を、
**管理者（YST・FTI）／本人／受入れ企業** がそれぞれのダッシュボードで管理できるWebアプリです。
**FTI・本人の画面はインドネシア語**、YST・企業は日本語（画面右上でいつでも切替可）。

ビルド不要の静的サイト（HTML/CSS/JS）なので **GitHub Pages でそのまま公開**でき、
共有データは **Supabase（無料）** に保存します。

---

## 現場課題への対応

| 課題 | このアプリでの対応 |
|---|---|
| ① SSW未合格者の **次回受験予定日** が未管理 | SSWタブに「次回受験予定日」欄。一覧にも表示し、未合格者を即把握 |
| ① 不合格時の **スコア** が未記録 | 受験ごとにスコアを記録。「合格までの距離」をバーで可視化 |
| ② 外免 **対策アプリの使用有無・頻度** が不明 | 「対策アプリ」欄でインストール有無・週あたり使用回数・最終使用日を管理。未使用者をKPI表示 |
| ② **模擬試験結果** が本人報告依存 | 模擬試験の日付・点数を記録し、推移をバーで表示 |
| ③ 日本語オンライン授業の **出席率** が未管理（最大の課題） | 荻野先生の週1授業を1回ずつ記録。**出席率を自動計算**し、60%未満をKPIで警告 |
| ③ 欠席理由・**自学状況** が不明 | 欠席理由を記録。本人も「自学学習ログ」を入力可能 |
| ③ 入国前の **知識確認対策** | 「知識確認対策（入国前）」のステータスを管理 |

## すぐ試す（デモモード）

Supabase設定なしでも、サンプルデータで全機能を確認できます（データはブラウザ内に保存）。

1. このフォルダの `index.html` をブラウザで開くだけ（ローカル確認）。
2. 役割を選んで「入る」。本人・企業はプルダウンで対象を選択。

## GitHub Pages で公開する手順

1. GitHubで新しいリポジトリを作成（例: `ssw-driver-tracker`、Public）。
2. このフォルダ内の全ファイルをアップロード（`index.html` がトップに来るように）。
   - Web画面なら「Add file ▸ Upload files」でフォルダごとドラッグ。
3. リポジトリの **Settings ▸ Pages** を開く。
4. **Source** を `Deploy from a branch`、**Branch** を `main` / `/(root)` にして Save。
5. 1〜2分後、`https://<ユーザー名>.github.io/ssw-driver-tracker/` が発行されます。このURLをリンク共有。

> `.nojekyll` を同梱しているので追加設定は不要です。

## 本番モード（共有DB＋全員で同期）にする手順

GitHub Pagesは静的サイトなので、データ共有は **Supabase** を使います（無料枠で十分）。

1. https://supabase.com でプロジェクトを作成。
2. **SQL Editor** で `schema.sql` を全文貼り付けて **Run**（テーブル＋サンプルデータ作成）。
3. **Project Settings ▸ API** から `Project URL` と `anon public` キーをコピー。
4. リポジトリの `config.js` を編集して貼り付け：
   ```js
   window.APP_CONFIG = {
     SUPABASE_URL: "https://xxxx.supabase.co",
     SUPABASE_ANON_KEY: "eyJhbGciOi...",
     ORG_NAME: "株式会社YSタレント"
   };
   ```
5. 保存・push すると、以後は全ユーザーのデータが同期されます（画面左下のバッジが「本番モード」に変わります）。

### セキュリティについて
`schema.sql` の初期設定は、URLを知る人のみが使う前提の簡易ポリシー（anonで読み書き）です。
社外秘データを扱う本番では、**Supabase Auth でのログイン**＋`authenticated`限定ポリシーへの変更を推奨します
（`schema.sql` 内にコメントで雛形あり）。ご希望あればログイン機能の追加実装も可能です。

## ファイル構成

```
ssw-driver-tracker/
├─ index.html      画面の入口
├─ config.js       Supabase接続設定（ここだけ編集すれば本番化）
├─ styles.css      デザイン
├─ i18n.js         日本語/インドネシア語 辞書
├─ data.js         データ層（Supabase or デモ）＋サンプルデータ
├─ app.js          画面ロジック（役割別ダッシュボード・各モジュール）
├─ schema.sql      DBテーブル・RLS・サンプルデータ
├─ .nojekyll
└─ README.md
```

## 役割ごとにできること

- **管理者（YST）**: 全候補者・全企業・全モジュールの閲覧と編集。課題KPIを一覧表示。
- **管理者（FTI／インドネシア語）**: 入国前の学習進捗（日本語・SSW・外免対策）の閲覧と編集。
- **本人（インドネシア語）**: 自分の進捗を確認。自学学習ログを入力。
- **受入れ企業**: 自社の候補者の進捗を閲覧。
